# Database ER Diagram

## Prompt

```text
Create an ER diagram for a SaaS billing system with users, teams, subscriptions, invoices, payments, and products.
```

## What to Expect

The diagram should show:

- Users belonging to teams
- Teams owning subscriptions
- Subscriptions connected to products or plans
- Invoices generated from subscriptions
- Payments linked to invoices
- Cardinality labels such as one-to-many where helpful

## Why This Demo Works

ER diagrams are one of the strongest use cases for AI-assisted diagramming because the user often knows the domain entities but does not want to manually arrange boxes and relationships from scratch.
